# 开发总计划

## 1. 目标

在基本架构确认后，进入分阶段开发。

原则如下：

1. 不一次性完成全部功能。
2. 每个子模块先出计划，再出详细文档。
3. 每个子模块文档都需要人工审查确认后，才能进入实现。
4. 实现阶段按确认后的文档逐步落地。

## 2. 开发方式

开发过程按“文档先行、逐步实现、阶段验收”的方式推进。

流程如下：

1. 先确定当前要做的子模块。
2. 输出该子模块的计划文档。
3. 输出该子模块的设计说明文档。
4. 你审查并修改文档。
5. 文档确认后再开始编码。
6. 完成后执行对应校验命令。
7. 进入下一个子模块。

## 3. 子模块拆分建议

建议优先按下列顺序拆分：

1. 前端工程基础与应用壳。
2. 主页项目卡片与创建导入。
3. 应用设置页。
4. 项目详情页。
5. 本地持久化数据模型。
6. Rust 后端命令与文件读写。
7. 接口文档页。
8. 接口测试页。
9. 主题系统与全局样式。

## 4. 当前阶段

当前阶段聚焦于主页项目卡片、创建导入和应用设置页的对齐与实现，不再使用旧的全局壳式路由方案。

## 5. 阶段验收标准

每个子模块在进入实现前，至少满足以下条件：

1. 范围清晰。
2. 输入输出清晰。
3. 与其他模块的依赖关系明确。
4. 风险点和不做事项明确。
5. 实现步骤可拆分。

## 6. 约束

1. 未确认的文档不进入编码。
2. 未完成审查的模块不并行扩展。
3. 每次只推进一个主子模块，避免范围失控。
