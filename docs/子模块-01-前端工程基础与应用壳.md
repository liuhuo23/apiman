# 子模块 01 - 前端工程基础与应用壳

> 历史文档：本文件描述的是早期全局壳方案，当前实现已切换为桌面应用式页面切换结构，不再作为后续开发依据。

## 1. 目标

建立前端最小可用基础层，为后续路由、页面和业务模块提供统一承载。

本子模块只关注工程壳和基础展示，不引入业务功能。

## 2. 范围

### 2.1 包含内容

1. 应用入口初始化。
2. 基础页面壳布局。
3. 全局样式和主题入口。
4. Ant Design 基础接入。
5. 预留路由挂载位置。
6. 预留后端命令调用封装位置。

### 2.2 不包含内容

1. 项目列表页面。
2. 项目详情页面。
3. 接口文档页面。
4. 接口测试页面。
5. 项目设置页面。
6. 本地文件持久化实现。
7. Rust 业务命令实现。

## 3. 当前输入输出

### 3.1 输入

1. 现有的 Tauri + React + TypeScript + Vite 工程。
2. 现有的 `antd` 依赖。

### 3.2 输出

1. 可承载后续页面开发的前端壳层。
2. 明确的目录分层约定。
3. 统一的基础视觉入口。

## 4. 目录建议

本子模块完成后，前端目录建议至少具备以下结构：

1. `src/pages/`：后续所有页面入口。
2. `src/components/`：通用组件。
3. `src/types/`：共享类型。
4. `src/services/`：后端调用封装。
5. `src/stores/`：全局状态。
6. `src/utils/`：工具函数。
7. `src/styles/`：全局样式和主题定义。

## 5. 页面壳职责

页面壳只做以下事情：

1. 提供统一窗口背景和基础布局。
2. 预留顶栏、侧栏、内容区或其他布局容器。
3. 承接后续路由出口。
4. 保持和桌面应用一致的视觉节奏。

页面壳不做以下事情：

1. 不承载具体业务逻辑。
2. 不直接处理项目数据。
3. 不直接实现接口测试或文档编辑能力。

## 6. 视觉约束

1. 页面必须有统一基础主题。
2. 明暗模式入口需要预留。
3. 基础颜色、边框、阴影和背景要可复用。
4. 组件视觉风格与桌面应用场景一致。

## 7. 依赖关系

本子模块依赖：

1. 现有前端工程可正常启动。
2. `antd` 基础样式可正常注入。

本子模块输出为后续模块提供依赖：

1. 路由模块。
2. 页面模块。
3. 状态管理模块。
4. 后端命令调用模块。

## 8. 实现步骤

在文档确认后，建议按以下顺序实现：

1. 梳理目录和文件边界。
2. 建立基础布局和主题入口。
3. 调整应用入口和根组件结构。
4. 预留后续路由和服务封装位置。
5. 完成基础校验。

## 9. 验收标准

该子模块完成后应满足：

1. 前端可以正常启动。
2. 页面壳可稳定承载后续模块。
3. 目录层次清晰。
4. 基础视觉规范已经落地。
5. 不影响后续功能扩展。

## 10. 校验命令

1. 执行 `pnpm build`。
