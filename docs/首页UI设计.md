# 首页 UI 设计说明

## 1. 设计目标

首页是应用入口，主要承担项目浏览、项目选择、项目创建和项目删除等核心动作。

首页 UI 需要满足以下目标：

1. 进入应用后第一眼看到项目列表。
2. 每个项目以 Card 形式独立呈现，便于快速扫描。
3. 支持单个项目进入详情页，也支持批量选择后执行管理操作。
4. 新建项目流程足够清晰，导入方式足够明确。
5. 删除操作可见但不误触。

## 2. 页面结构

首页建议分为以下区域：

1. 顶部标题区。
2. 工具区。
3. 项目列表区。
4. 创建项目模态框。
5. 删除确认交互。

### 2.1 顶部标题区

顶部标题区用于表达当前页面身份和全局状态。

建议包含：

1. 页面标题。
2. 简短说明文字。
3. 当前项目数量或状态提示。

### 2.2 工具区

工具区用于放置高频操作。

建议包含：

1. 创建项目按钮。
2. 批量操作入口。
3. 搜索或筛选入口，后续可扩展。

工具区布局应保持清晰，不与项目列表抢占视觉优先级。

### 2.3 项目列表区

项目列表使用 Card Grid 呈现，每个项目一个独立 Card。

每个 Card 必须包含：

1. 项目名称。
2. 创建时间。
3. 最新更新时间。
4. 右下角删除图标。
5. 选中状态标识。

每个 Card 的主行为是点击进入项目详情页。

Card 内部建议采用以下层级：

1. 标题区。
2. 时间信息区。
3. 状态和辅助信息区。
4. 右下角操作区。

### 2.4 创建项目模态框

点击创建按钮后弹出模态框。

模态框需要支持三种方式：

1. URL 导入。
2. 文件上传导入。
3. 跳过导入，直接创建空项目。

模态框内建议分为以下内容：

1. 导入方式切换区。
2. 当前方式的表单内容区。
3. 底部操作按钮区。

## 3. 导入方式设计

### 3.1 URL 导入

用于输入标准 Swagger / OpenAPI 地址。

建议包含：

1. URL 输入框。
2. 解析提示文本。
3. 可选的校验反馈。

### 3.2 文件上传导入

当前仅支持以下格式：

1. `openapi.json`
2. `yaml`

建议包含：

1. 文件上传控件。
2. 支持格式说明。
3. 上传后解析状态展示。

### 3.3 跳过导入

允许用户直接创建空项目。

适用场景：

1. 用户暂时没有 OpenAPI 文件。
2. 用户先创建项目骨架，后续再补文档。

## 4. 删除交互设计

删除入口位于 Card 右下角，使用图标表达。

删除交互规则：

1. 删除图标要明确可见，但不能抢主操作。
2. 删除动作和进入详情页动作必须分离。
3. 删除前需要二次确认。
4. 多选后可支持批量删除。

建议删除确认使用独立确认弹窗，而不是直接执行。

## 5. 多选设计

首页支持卡片多选，用于后续批量操作。

多选交互建议：

1. 单击卡片主体进入详情。
2. 使用卡片选择状态或快捷操作进入多选。
3. 多选后显示批量操作条。
4. 批量操作仅对已选中的项目生效。

多选状态下要避免与“进入详情”冲突。

## 6. 状态设计

首页至少需要以下状态：

1. 项目列表数据。
2. 选中的项目集合。
3. 创建模态框显示状态。
4. 当前导入方式。
5. 删除确认状态。
6. 加载与错误状态。

## 7. 组件拆分建议

首页建议拆分为以下组件：

1. `HomePage`：页面容器。
2. `ProjectToolbar`：顶部工具区。
3. `ProjectGrid`：项目列表容器。
4. `ProjectCard`：单个项目卡片。
5. `CreateProjectModal`：创建项目弹窗。
6. `ImportMethodSwitcher`：导入方式切换。
7. `DeleteConfirmDialog`：删除确认弹窗。
8. `BatchActionBar`：批量操作条。

## 8. 视觉规范

首页视觉风格需要满足：

1. 卡片层次清晰。
2. 删除按钮显眼但克制。
3. 选中状态明确可识别。
4. 创建弹窗层级高于页面内容。
5. 不同导入方式有明确分区感。

建议使用：

1. Ant Design 的 Card、Modal、Button、Checkbox、Tooltip、Dropdown、Tag。
2. React 常规事件处理方式。
3. 一致的圆角、阴影和边框规范。

## 9. 交互流程

### 9.1 打开首页

1. 进入应用后显示项目列表。
2. 若项目较少，优先展示卡片信息完整性。
3. 若项目较多，后续再补搜索和筛选。

### 9.2 创建项目

1. 点击创建按钮。
2. 弹出创建模态框。
3. 选择导入方式。
4. 完成导入或跳过导入。
5. 创建成功后刷新列表。

### 9.3 进入详情

1. 点击项目 Card 主体。
2. 跳转到项目详情页。
3. 保留上次浏览状态的能力可放在后续阶段处理。

### 9.4 删除项目

1. 点击 Card 右下角删除图标。
2. 弹出确认弹窗。
3. 确认后删除项目。
4. 刷新列表并同步选中状态。

## 10. 验收标准

该设计文档通过后，应满足：

1. 首页作为应用入口的职责明确。
2. 项目卡片的展示信息明确。
3. 创建项目流程明确。
4. 导入方式明确。
5. 删除与多选交互明确。
6. 可以进入实现阶段。
